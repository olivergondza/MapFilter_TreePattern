Generator example

Following example shows structure of parser for fictive configuration file of utility to generate some outcome of its input files.

There is a mandatory input attribute that holds either single or array of input values. The other common mandatory attribute is output_format that defines either one or several output formats to generate, a whitelist for allowed output types and the default output type.

There are some mandatory attributes that are utilized with certain output types. Whereas the dtd attribute is specific for the xml output type the stylesheet attribute is allowed for html output type too. An extension is an attribute dedicated to man output format.

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE pattern SYSTEM "TreePattern053.dtd">
<pattern>
  <all>

    <key name="input" assert="no_source">
      <iterator />
    </key>

    <key_attr attr="output_format" iterator="yes" default="html" valuePattern="html|man|xml">

      <attr forValue="html|xml" default="default.css" valuePattern="/\.css$/" iterator="yes">
          stylesheet
      </attr>

      <key forValue="man" name="extension">
        <value pattern="/^[1-8]$/" default="3" />
      </key>

      <key forValue="xml" name="dtd">
        <value pattern="/\.dtd$/" default="default.dtd" />
      </key>

    </key_attr>

  </all>
</pattern>

There is a list of input arrays in the first row of each set and the output array obtained by MapFilter_TreePattern::getResults() in the second one.
