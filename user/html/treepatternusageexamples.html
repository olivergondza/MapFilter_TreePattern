TreePattern Usage Examples

Whitelists

Exclusive Whitelist

This simple example shows how to create simple whitelist allowing occurrence of just one of defined attributes. my_utility -h -v -o a.out

        <pattern>
          <one>
            <key name="-h" />
            <key name="-v" />
          </one>
        </pattern>

Filtered query now contains just -h option since patterns prevents occurence of not declared options and enforces existence of either -v or -h. Whenever -v and -h passed simultaneously -h is picked since it is declared earlier in the one element.

Simple Whitelist

Let's assume the same query with slightly modified pattern.

        <pattern>
          <opt>
            <key name="-h" />
            <key name="-v" />
          </opt>
        </pattern>

This is the way to enable occurrence of more options (including 0). The same query will now contain both -h and -v option regardless of their order.

Full Sets

Sometimes it's useful to specify a group of attributes that all must occur simultaneously. my_app -f main.c -o a.out

        <pattern>
          <all>
            <key name="-f" />
            <key name="-o" />
          </all>
        </pattern>

Since we have got all the arguments, they all pass our filter. Let's try to run the binary without source file. my_app -o a.out

A filtered query would be empty because our pattern demands occurrence of both (all) options.
